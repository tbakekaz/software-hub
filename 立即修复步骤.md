# 🚀 立即修复 522 错误 - 3 步操作

## ⚡ 快速操作（5 分钟）

### 步骤 1：进入 Cloudflare Dashboard（1 分钟）

1. 打开：https://dash.cloudflare.com
2. 登录你的账号
3. 点击：**Workers 和 Pages** → **software-hub** → **设置**（Settings）

### 步骤 2：启用 nodejs_compat 标志（2 分钟）

在设置页面找到 **"Functions"** 或 **"兼容性标志"** 部分：

1. **生产环境**（Production）：
   - 找到 "兼容性标志"（Compatibility Flags）
   - 点击 "添加标志"（Add Flag）或 "+"
   - 输入：`nodejs_compat`
   - 点击保存

2. **预览环境**（Preview）：
   - 同样添加 `nodejs_compat` 标志
   - 保存

### 步骤 3：重新部署（2 分钟）

1. 点击 **"部署"**（Deployments）标签
2. 找到最新的部署
3. 点击 **"重新部署"**（Retry deployment）按钮
4. 等待 5-10 分钟

## ✅ 完成！

部署完成后，访问：
- https://software-hub.pages.dev
- https://kazsoft.dpdns.org

应该可以正常访问了！

## 📋 如果找不到兼容性标志设置

如果设置页面中没有看到兼容性标志选项：

1. **尝试通过 Functions 设置**：
   - 在设置页面找到 "Functions" 部分
   - 查看是否有 "兼容性标志" 选项

2. **检查项目类型**：
   - 确保这是 Pages 项目，不是 Workers 项目

3. **使用 API 设置**（高级）：
   - 可以使用 Cloudflare API 直接设置
   - 或联系 Cloudflare 支持
